# Image Generation Prompts for ElBruno.MCPToolRouter

This document contains detailed prompts for generating marketing and branding images for the ElBruno.MCPToolRouter project. Each section includes DALL-E prompts, technical specifications, and visual guidelines.

---

## 1. NuGet Package Logo (128x128 / 256x256)

**Purpose:** Package icon displayed on NuGet.org and in IDE package managers  
**Aspect Ratio:** 1:1 (square)  
**Recommended Resolution:** 256x256 pixels (scale to 128x128 for display)

### Visual Concept
A minimal, iconic representation of semantic routing and tool selection. The design should be:
- **Readable at small sizes** (icon will scale down to 64x64)
- **Clean and modern** — no gradients or excessive detail
- **Color-coded** to suggest AI/routing theme
- **Metaphorical** — convey tools, routing, and intelligence

### DALL-E Prompt

```
Create a minimalist tech icon (256x256 px) showing semantic routing and tool selection.
Design a logo featuring:
- A central circular node or hub (representing MCPToolRouter)
- 3-4 arrows or vector lines radiating outward in different directions
- Each arrow pointing to simple tool symbols (wrench, gear, key, magnifying glass)
- Color scheme: Deep blue (#0066CC) for the hub, electric purple (#9D4EDD) for routing arrows
- Accents in cyan (#00D9FF) for highlights on tool symbols
- Clean, outlined style (not filled) with 2-3 pixel line weight
- Flat design (no 3D depth or shadows)
- White or transparent background
- Modern, tech-forward aesthetic suitable for .NET/C# community
- No text or labels

The overall composition should fit perfectly in a square with even margins (10-20px padding).
Style reference: Modern icon design from Apple, GitHub, and Microsoft design systems.
```

### Color Palette
| Color | Hex | Usage |
|-------|-----|-------|
| Primary Blue | `#0066CC` | Hub/center node |
| Electric Purple | `#9D4EDD` | Routing arrows |
| Cyan Accent | `#00D9FF` | Tool highlights |
| Dark Gray | `#1a1a1a` | Outlines/strokes |
| White | `#FFFFFF` | Background / negative space |

### Key Visual Elements
✅ Central hub representing the router  
✅ Radiating arrows showing routing/distribution  
✅ Multiple tool symbols (wrench, gear, key, etc.)  
✅ Clean, outlined vector style  
✅ Professional, scalable design  

### What to Avoid
❌ Photorealism or complex textures  
❌ Gradients or multi-color fills  
❌ Drop shadows or 3D effects  
❌ Text or labels inside the icon  
❌ Excessive detail (must be recognizable at 64x64)

## 2. YouTube Video Thumbnail (1920x1080)

**Purpose:** Eye-catching thumbnail for YouTube videos about MCPToolRouter  
**Aspect Ratio:** 16:9 (widescreen)  
**Recommended Resolution:** 1920x1080 pixels  

### Visual Concept
A bold, high-contrast thumbnail that stands out in YouTube feeds. Should immediately convey:
- **The benefit:** Token savings, efficiency, speed
- **The product:** MCPToolRouter
- **The context:** LLM tools, AI, .NET

### DALL-E Prompt

```
Create a striking YouTube video thumbnail (1920x1080 px, 16:9 aspect ratio).

Design composition:
- Left side (40%): A stylized illustration of tools being sorted/filtered through a magical funnel or prism
  - Show 10-15 tool icons (wrench, hammer, database, key, binoculars, folder, code, etc.)
  - Some tools are highlighted in cyan/neon, others faded in the background
  - Visual motion/flow suggesting filtering and selection
  
- Right side (60%): Bold typography and key messaging
  - Text overlay (large, bold, sans-serif): "Save 80% Tokens" or "Smart Tool Routing"
  - Subtitle in smaller text: "with MCPToolRouter"
  - Add an arrow or chevron pointing right, suggesting progress/improvement
  
- Color scheme: Dark modern gradient background (dark blue to black: #001a4d → #000000)
  - Bright neon cyan (#00D9FF) for highlighted tools and text
  - Purple accents (#9D4EDD) for secondary elements
  - White text for maximum contrast and readability
  
- Visual style: 3D isometric or modern flat design with subtle depth
- No gradients on text (solid white/cyan only)
- High contrast for readability on small screens
- .NET branding: subtle C# or .NET logo in corner if needed
- Professional, modern, energetic vibe

Elements to include:
- Animated-style motion lines suggesting speed and efficiency
- Clear focal point (text area) with minimal visual competition
- Room for video title overlay in bottom half
- Balanced composition that works at thumbnail size (160x90 px)
```

### Color Palette
| Color | Hex | Usage |
|-------|-----|-------|
| Dark Blue Background | `#001a4d` | Primary background (left side) |
| Black Background | `#000000` | Primary background (right side) |
| Electric Cyan | `#00D9FF` | Highlighted tools, accents |
| Electric Purple | `#9D4EDD` | Secondary elements |
| White | `#FFFFFF` | Text, contrast |
| Dark Gray | `#333333` | Faded tools background |

### Key Visual Elements
✅ Tool filtering/funnel metaphor  
✅ Clear, bold text overlay  
✅ High contrast for readability  
✅ Motion/flow suggesting efficiency  
✅ Highlights and secondary tools showing filtering  

### What to Avoid
❌ Small text (must be readable at thumbnail size)  
❌ More than 2 main text blocks  
❌ Busy backgrounds that compete with text  
❌ Muted colors (needs to pop in a YouTube feed)  
❌ Distracting elements outside focal point

## 3. Blog Post Header (1200x630)

**Purpose:** Social card and blog post header image  
**Aspect Ratio:** 16:9 (widescreen)  
**Recommended Resolution:** 1200x630 pixels  

### Visual Concept
A professional, clean header that works well with text overlay and communicates the core value prop. Should feel:
- **Authoritative and polished** — suitable for a blog
- **Minimalist** — doesn't compete with blog title text
- **On-brand** — consistent with product and .NET community aesthetics

### DALL-E Prompt

```
Create a professional blog post header image (1200x630 px, 16:9 aspect ratio).

Design composition:
- Background: Subtle gradient (deep blue #0066CC fading to purple #9D4EDD) with minimal grain texture
  - Keep background simple to allow text overlay (blog title, subtitle)
  - Gradient should be smooth, not harsh
  
- Center focal element (takes up ~60% of width):
  - Modern illustration of a vector/network diagram
  - Show 8-10 circular nodes representing tools, connected by flowing lines
  - Some nodes highlighted in bright cyan (#00D9FF), others in lighter blue
  - Lines showing the "routing" or "filtering" of tools towards a central point
  - Lines can have subtle glow or gradient effect
  - 3D isometric view or clean flat design style
  
- Visual metaphors to include:
  - Magnifying glass overlay on some tool nodes (suggesting search/semantic routing)
  - Checkmark or highlight on selected tools
  - Faded/reduced opacity on non-selected tools
  
- Right edge (20%): Negative space for text overlay
  - Pure white or very light gray background
  - This is where blog title and date will be placed
  
- Visual style: Modern, clean, minimalist
  - No overly complex patterns or textures
  - Subtle shadows/depth but not 3D rendering
  - Consistent line weights and proportions
  
- Branding: Optional small ElBruno or MCPToolRouter text in bottom-right corner (faint, 20% opacity)

Elements to include:
- Multiple tool representations (diverse icons: database, API, key, folder, etc.)
- Clear visual hierarchy showing routing/filtering
- Emphasis on intelligence/semantic understanding
- Professional, trustworthy aesthetic suitable for tech blog
```

### Color Palette
| Color | Hex | Usage |
|-------|-----|-------|
| Deep Blue | `#0066CC` | Background gradient start |
| Electric Purple | `#9D4EDD` | Background gradient end |
| Cyan Accent | `#00D9FF` | Highlighted nodes |
| Light Blue | `#4DA6FF` | Secondary nodes |
| White | `#FFFFFF` | Text overlay area |
| Dark Gray | `#2a2a2a` | Fine details/lines |

### Key Visual Elements
✅ Node/network diagram showing tool routing  
✅ Semantic search visualization (magnifying glass)  
✅ Clear distinction between selected and non-selected tools  
✅ Smooth gradient background  
✅ Negative space for text overlay  

### What to Avoid
❌ Busy patterns that distract from text overlay  
❌ Overly complex network diagrams  
❌ Hard shadows or harsh lighting  
❌ Text on gradient background (use text area instead)  
❌ Overly trendy design that dates quickly

## 4. Twitter / LinkedIn Social Card (1080x1080)

**Purpose:** Shareable social media card for Twitter and LinkedIn  
**Aspect Ratio:** 1:1 (square)  
**Recommended Resolution:** 1080x1080 pixels  

### Visual Concept
A bold, shareable design that works in social feeds. Should communicate value in 2-3 seconds and encourage shares/engagement.

### DALL-E Prompt

```
Create a bold social media card (1080x1080 px, 1:1 square aspect ratio) for Twitter and LinkedIn.

Design composition:
- Bold, contrasting background using deep blue to purple gradient (#0066CC → #9D4EDD)
  - Can include a subtle diagonal stripe or accent pattern (but not busy)
  - Ensure background is cohesive and doesn't distract from text
  
- Centered visual element (45% of card):
  - Stylized illustration showing tool routing/semantic search
  - Simple, iconic representation: 3-4 circles (tools) in a row, with an arrow or flow pointing to 1-2 highlighted circles
  - Tool symbols inside circles (wrench, gear, key, database, etc.)
  - Highlighted circles in bright cyan (#00D9FF)
  - Non-highlighted circles in light gray or secondary color
  - Clean, modern line style with 3-4 pixel strokes
  
- Text overlay (35% of card):
  - Bold headline: One of these options:
    * "Smart Tool Routing"
    * "Semantic Search for MCP Tools"
    * "Save Tokens with MCPToolRouter"
    * "Intelligent Tool Selection"
  - Subtext (smaller): "Open-source .NET library"
  - Font: Bold sans-serif (similar to Montserrat, Poppins, or Inter)
  - Text color: White (#FFFFFF) for maximum contrast
  - No drop shadows on text, but ensure legibility
  
- Bottom branding (10%):
  - Small ElBruno logo or "ElBruno.MCPToolRouter" text
  - Light gray or white text, 60-70% opacity
  - GitHub repo URL (optional): github.com/elbruno/ElBruno.ModelContextProtocol
  
- Style: Energetic, modern, professional
  - Flat design with minimal depth
  - High color saturation for engagement
  - Bold, clean typography
  
- Key design principle: Every element should be recognizable when the card scales down to 300x300px (mobile preview)
```

| White | `#FFFFFF` | Text |
| Light Gray | `#CCCCCC` | Secondary elements |
| Dark Gray | `#333333` | Borders/outlines |

### Key Visual Elements
✅ Tool filtering/routing visualization  
✅ Bold, shareable headline  
✅ High contrast text for readability  
✅ Clean branding  
✅ Energetic, modern aesthetic  

### What to Avoid
❌ Cluttered designs that don't scale to thumbnail  
❌ Small text (must be readable at 300x300)  
❌ More than 2 main visual focal points  
❌ Busy background patterns

## Usage Guidelines

### Ordering Images
When commissioning these images:
1. **Priority Order:**
   - NuGet Package Logo (essential for NuGet listing)
   - Blog Post Header (supports documentation/marketing)
   - YouTube Thumbnail (for video content)
   - Social Card (amplifies reach)

2. **Delivery Format:**
   - PNG with transparent background (logo)
   - PNG or JPG for all others
   - Provide both recommended resolution and 2x scale (for high-DPI displays)

3. **File Naming:**
   - `nuget-logo-256.png` → `images/nuget_logo.png`
   - `blog-header-1200x630.png` → `images/blog-header.png`
   - `youtube-thumbnail-1920x1080.png` → `images/youtube-thumbnail.png`
   - `social-card-1080x1080.png` → `images/social-card.png`

### Brand Consistency
All images should share:
- **Color palette:** Blue (#0066CC), Purple (#9D4EDD), Cyan (#00D9FF)
- **Typography:** Bold, modern sans-serif
- **Style:** Flat design with subtle depth, no gradients or photorealism
- **Messaging:** Focus on semantic routing, tool efficiency, and AI intelligence
- **Audience:** .NET developers, LLM/AI practitioners

### Customization Notes
- **ElBruno branding:** Ensure "ElBruno" or project name appears on larger images (blog header, social card)
- **MCP context:** If space allows, reference "Model Context Protocol" or "MCP tools"
- **Version updates:** If the project pivots or major features are added, these prompts should be revisited
